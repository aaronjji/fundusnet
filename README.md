Extends: SkinToneNet (TMLR June 2025)

## Quickstart

### 1. Get EyePACS data
```bash
pip install kaggle
# Put kaggle.json in ~/.kaggle/
kaggle competitions download -c diabetic-retinopathy-detection
unzip diabetic-retinopathy-detection.zip -d data/
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Quick sanity check (500 images, ~5 minutes)
```bash
python run_preprocessing.py --step all --debug
```

### 4. Full pipeline (~2-3 hours on 8 CPU cores)
```bash
python run_preprocessing.py --step all
```

### 5. Check the key result
```bash
cat results/snr_analysis/snr_summary.json
```

Look for `snr_ratio_light_over_dark`. 
- `> 3.0×` → Strong mechanistic support, start training
- `1.5–3.0×` → Moderate support, fundus effect smaller than dermoscopy (expected, still valid)
- `< 1.5×` → Review ODCR proxy before proceeding

## Key Design Decisions

### Why patient-level splits?
Image-level splits inflate test AUC by 3–8 points because both eyes of the same
patient appear in train and test. SkinToneNet Section 3.1 documents this.

### Why ODCR instead of ground-truth ethnicity?
EyePACS has no ethnicity labels. ODCR is computed from the peripapillary region
in L*a*b* space — the same optical proxy approach as ITA in SkinToneNet.
This is a methodological contribution: the first validated fundus pigmentation
proxy for fairness analysis.

### Why the green channel for SNR?
Microaneurysms and haemorrhages absorb strongly in the green channel (peak ~550nm),
maximising lesion-background contrast. The same physical reason clinical fundus
photography uses green-channel emphasis.

## Locked Test Set

`data/splits/test.csv` is locked after generation.
Do not examine per-subgroup metrics on the test set until all ablation variants
are trained. This mirrors SkinToneNet's methodology exactly.
Pre-specify all three permutation tests before unblinding.
